# z-skill Design System

> 版本：0.5
> 建立日期：2026-07-13
> 适用范围：z-skill 品牌首页、工具目录、工具详情页、About 页面及后续同类公开页面
> 当前状态：正式四页面已按本规范完成首轮重构并部署，当前进入发布机制标准化阶段

## 1. 设计目标

z-skill 是周全个人专属的 AI 工具发布站。设计必须帮助访问者快速回答四个问题：

1. 这里发布什么；
2. 当前有哪些真实工具；
3. 工具是否适合自己的场景；
4. 怎样安装或下载。

网站不是社区、排行榜、文章站或后台管理系统。视觉应当简洁、克制、专业，同时具备清晰的个人品牌识别度。设计感主要来自比例、排版、留白、边界和少量蓝色强调，不依赖大面积渐变、插画堆叠或重阴影。

### 1.1 awesome-design-md 深度参考记录

本规范使用 `VoltAgent/awesome-design-md` 的 Mintlify、Vercel 和 Intercom 三份完整样本进行交叉分析。样本只用于提取通用设计机制，不作为视觉模板。

| 样本 | 采用的通用机制 | 明确排除的品牌特征 | 转化为 z-skill 的规则 |
|---|---|---|---|
| Mintlify | 宽松的品牌首页与高密度技术详情分层；正文与代码字体分工；组件状态和响应式规则完整记录 | 薄荷绿、云层渐变、插画英雄区、Inter/Geist 专有组合 | 首页保持宽松，详情页收紧信息密度；安装 Prompt 使用深色等宽面板；中文继续使用系统字体 |
| Vercel | 4px 间距、精确灰阶、细边框、技术标签等宽化、叠层轻阴影 | 彩色网格渐变、Geist 品牌身份、黑白品牌主色、标志性超大胶囊 CTA | 技术眉题和纯版本号使用系统等宽字体；卡片以 hairline 和轻叠层阴影分级；品牌主色仍为 z-skill 蓝 |
| Intercom | 页面底色与白卡形成表面差；产品内容优先；克制圆角；避免无意义装饰 | 奶油色画布、Fin 橙、Saans 字体、产品截图主导的品牌节奏 | 保留冷灰画布与白卡层级；页面内容以真实工具、测试和安装信息为主；不引入第二强调色 |

共同规则进入 z-skill 前必须同时满足：不依赖第三方专有字体、不携带可识别品牌资产、符合个人发布站定位、能改善中文阅读或工具信息交付。任何单一样本的标志性颜色、渐变、按钮形状和营销文案都不得直接复制。

## 2. 品牌气质

关键词：`清醒`、`可靠`、`可验证`、`可迁移`、`个人策展`。

- 清醒：信息直接，不制造热闹感和平台规模感。
- 可靠：版本、测试、隐私、限制与下载入口处于同一信息体系。
- 可验证：状态标签只描述真实完成度，不用评分或模糊背书替代证据。
- 可迁移：安装 Prompt 与 ZIP 下载并列，兼顾 Agent 与人工使用。
- 个人策展：明确由周全设计、整理或验证，不伪装成开放平台。

文字语气：短句、事实优先、少形容词。避免“颠覆”“赋能”“海量”“顶级”“一键起飞”等营销措辞。

## 3. Logo 与 Z 字标志

### 3.1 组合标志

标准组合由蓝色 Z 字标志和 `z-skill` 字标组成：

- Z 标志在左，字标在右；
- 标准间距为 10px；
- 导航中的组合标志总高度为 34px；
- 字标使用页面主字体，20px / 750，字距 `-0.04em`；
- 不在组合标志旁增加 slogan、版本号或状态点。

### 3.2 Z 字标志

- 容器：34×34px，圆角 10px；紧凑场景可用 30×30px，圆角 8px；
- 背景：品牌蓝 `#3157D5`；
- 字符：大写 `Z`，白色，20px / 800，字距 `-0.07em`；
- 几何居中后向左 0.3px、向上 0.5px进行光学校正；
- CSS 实现应使用内部独立元素应用 `transform: translate(-0.3px, -0.5px)`，不要移动外层容器；
- 24px 以下不得继续缩小完整 Z 标志，应改用纯文字字标。

禁止：使用 SkillHub 的 Logo、吉祥物或轮廓；使用渐变背景；把 Z 拉伸、旋转或加投影；同时出现大写 `Z-SKILL` 与标准字标。

## 5. 字体与排版

### 5.1 字体栈

不加载第三方专有字体。默认使用系统字体：

```css
font-family: ui-sans-serif, -apple-system, BlinkMacSystemFont,
  "Segoe UI", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
  "Noto Sans CJK SC", sans-serif;
```

代码与安装 Prompt：

```css
font-family: "SFMono-Regular", Consolas, "Liberation Mono", monospace;
```

技术眉题、纯版本号、文件格式和命令使用同一系统等宽字体；中文正文、导航和按钮不使用等宽字体。该分工用于提示“这是技术信息”，不用于模仿任何第三方品牌字体。

### 5.2 字号层级

| 层级 | 桌面 | 手机 | 行高 | 字重 |
|---|---:|---:|---:|---:|
| Display | 64px | 42px | 1.06 | 720 |
| H1 | 48px | 36px | 1.12 | 720 |
| H2 | 32px | 27px | 1.22 | 680 |
| H3 | 20px | 19px | 1.35 | 650 |
| Lead | 18px | 17px | 1.75 | 400 |
| Body | 15px | 15px | 1.78 | 400 |
| Small | 13px | 13px | 1.65 | 400 |
| Label | 11px | 11px | 1.4 | 700 |

中文正文不使用过度负字距。仅拉丁字母标题、Logo 与数字可使用 `-0.02em` 至 `-0.05em`。正文每行建议 32—40 个中文字符，说明文字不超过 46 个。

## 6. 间距、栅格与页面宽度

基础间距单位为 4px。优先使用：`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64 / 80 / 96`。

- 主内容最大宽度：1200px；
- 阅读内容最大宽度：760px；
- 桌面页边距：32px；平板：24px；手机：16px；
- 桌面 12 栏栅格，栏间距 24px；
- 详情页正文与侧栏建议为 8:4，侧栏宽度 300—340px；
- 页面主要区块垂直间距：80—96px；手机：56—64px；
- 卡片内部间距：24px；紧凑列表：20px；手机：18px。

### 6.1 页面密度与表面节奏

- 品牌首页属于营销阅读面：首屏与主要区块使用80—96px节奏，突出定位、搜索和当前代表工具；
- 工具目录属于检索面：控制区与结果列表压缩到12—24px节奏，提高扫读效率；
- 工具详情属于技术阅读面：正文保持15—16px舒适行距，版本、格式、测试与Prompt使用更紧凑的技术层级；
- `--canvas` 承担页面底色，`--surface` 承担导航、卡片和面板；层级优先通过表面差与hairline表达，不靠大阴影；
- 深色表面只用于安装Prompt和Footer等明确功能区，不把整页做成深色开发者模板。

## 7. 圆角、边框与阴影

- 小控件：8px；
- 输入框、按钮：10px；
- 常规卡片：14px；
- 大面板、搜索框：18px；
- 胶囊标签：999px；
- 常规边框：1px `--line-200`；
- 卡片默认不用阴影；品牌首页代表工具或悬浮状态可使用 `0 1px 2px rgb(16 21 34 / 0.04), 0 10px 30px rgb(16 21 34 / 0.05)` 的轻叠层；
- 叠层阴影只允许两个低透明度层级，不使用单层厚重阴影、品牌色发光或外发光；焦点环除外。

## 8. 导航

- 桌面高度 72px，白色或 96% 白色背景，下边框 1px；
- 左侧为品牌，右侧为 `首页 / 全部工具 / 关于`；
- 当前页用深色文字与 2px 蓝色短下划线，不用大块填充胶囊；
- 主导航不按 Skill、Workflow、Agent 拆入口，这些属于目录筛选；
- 当前页必须设置 `aria-current="page"`；页面首个键盘焦点提供“跳到主要内容”；
- 手机端保留三个文字入口，空间不足时允许横向滚动，不使用尚未实现的汉堡菜单；
- 导航不出现登录、投稿、排行榜或“持续在线”等平台化状态。

## 9. 品牌首页

- 首页与目录分离；首屏只承担定位、搜索和当前代表工具入口；
- Display 标题最多两行；副标题明确“由周全设计、整理并验证”；
- 首页主文案优先说明“怎样帮助访问者理解和取得工具”，不把禁止事项或防御性声明当作首屏主卖点；
- 大搜索框是首屏主要交互，高度 60—64px，带明确 placeholder 与搜索按钮；
- 搜索提交后进入目录并保留关键词；
- 可展示“当前公开 1 项”这类可核验事实，但不展示下载量、用户数、评分或增长数据；
- 首屏可使用一条蓝色结构线或小型网格作为品牌识别，不堆叠装饰性渐变。

## 10. 搜索框

- 首页大搜索：64px 高，18px 圆角，正文 16px；
- 目录搜索：44px 高，10px 圆角，正文 14px；
- 搜索图标应简洁，优先使用 CSS 或现有图标库，不引入品牌图形；
- Focus：边框变为品牌蓝，并显示 3px、12% 透明度焦点环；
- 无结果时说明“没有匹配的公开工具”，并提供清除条件按钮；
- 不提供没有真实行为的搜索、排序或视图切换控件。

## 11. 工具目录与工具卡片

### 11.1 目录控制区

- 类型、状态与排序都必须产生真实结果；
- 当前只有一个工具时，可保留有效筛选，不显示虚假的“热门”“飙升”“下载排行”；
- 默认排序为“最近更新”；支持项必须能在数据增多后确定性排序；
- 结果数量必须来自真实数据。

### 11.2 工具卡片

- 单工具阶段默认采用全宽列表卡，避免双列布局留下“尚未填满”的模板空位；工具数量增长后再复验单列与双列密度；
- 卡片层级：图形标识 → 名称与状态 → 一句话用途 → 已验证格式 → 版本与更新时间 → 操作；
- 整张卡片可进入详情页，但下载使用独立按钮；
- 主操作为“查看详情”，下载为次操作，避免误下载；
- Hover：边框变蓝灰，向上移动 2px，可增加单层轻阴影；
- 卡片内不得出现评分、收藏、下载次数或虚构兼容环境。

## 12. 工具详情页

详情页首屏应一次呈现：名称、类型、状态、用途、版本、作者与主操作。正文顺序建议：

1. 核心用途；
2. 适用场景；
3. 已验证格式；
4. 使用说明；
5. Agent 安装 Prompt；
6. 下载与版本；
7. 许可证、隐私、已知限制和测试状态。

正文使用阅读列，重要发布信息使用右侧下载面板。手机端侧栏下移至正文关键说明之后，安装 Prompt 必须先于重复下载入口。

长详情页在正文顶部提供核心用途、测试、使用、安装与限制的页内导航；导航可吸附且在手机端横向滚动，不额外占用多行首屏。未发布 slug 使用z-skill自己的404页面，不暴露框架默认页面。

## 13. 安装 Prompt 面板

- 面板使用深色背景 `#111827`，与全站浅色页面形成单一强对比；
- 标题区包含“复制给 Agent”、用途说明和复制按钮；
- Prompt 使用等宽字体，14px / 1.7，允许换行，不横向挤压中文；
- 内容必须是可执行的真实提示，不承诺尚未验证的自动安装结果；
- Prompt 必须包含可直接访问的绝对下载地址或权威安装清单地址，并标明来源版本；不得要求 Agent 自行猜测下载位置；
- 默认按钮文字“复制 Prompt”；成功后变为“已复制”，同时出现可被读屏器读取的状态文本；
- 成功状态保持约 2 秒，再恢复默认；
- Clipboard API 不可用时选中文本并提示用户手动复制，不静默失败。

## 14. 下载面板

- 包含版本、发布状态、作者、许可证、文件类型、验证环境、SHA-256、隐私提示和下载按钮；
- 不硬编码未经复核的文件大小；
- 主按钮文案包含真实对象，如“下载 ZIP”；
- 下载按钮与安装 Prompt 并列为两条路径，不能让 ZIP 成为唯一使用方式；
- 隐私说明位于下载按钮上方或紧邻位置，避免下载后才看到。

## 15. 状态标签

- 标签只表达一种维度；类型标签与发布状态标签分开；
- 高度 24px，左右内边距 9px，11px / 700；
- “公开候选”使用琥珀色；“已验证”使用绿色；“尚未验证”使用中性灰；
- 不用红色表示“尚未验证”，红色仅留给明确风险；
- 已明确发生的测试失败使用风险红，与“尚未验证”的中性灰区分；
- 标签不能替代正文中的测试依据。

## 16. About 页面

About 不是品牌宣传长页，应解释：

- z-skill 发布什么、不发布什么；
- 收录标准与验证原则；
- 候选版、已验证等状态如何理解；
- 公众号、GitHub 与网站的分工；
- 作者是谁，以及如何判断信息时效。

页面采用短章节与编号原则卡，不展示虚构统计数字、社区入口、二维码占位或未确定的联系方式。

## 17. Footer

- 深色背景，建立明确收尾；
- 左侧为 `z-skill` 与一句定位；右侧为首页、全部工具、关于；
- 可补充“版本、隐私与限制以工具详情页为准”；
- 不复制主导航全部分类，不放登录、社区、投稿与未启用社交链接；
- 手机端纵向排列，链接触控高度不低于 44px。

## 18. 交互状态

### Hover

- 链接：颜色转品牌蓝；
- 主按钮：背景转 `--blue-700`，不放大；
- 次按钮：边框转 `--blue-500`；
- 卡片：上移 2px，并增加本规范允许的轻叠层阴影。

### Focus

所有链接、按钮、输入框和选择框必须支持 `:focus-visible`：

```css
outline: 2px solid var(--surface);
outline-offset: 0;
box-shadow: 0 0 0 5px var(--blue-700);
```

白色内环用于深色表面，深蓝外环用于浅色表面；不得使用低透明度焦点环，也不得只依赖颜色变化表示焦点。搜索框由完整容器显示同等级焦点环，内部无边框输入框不重复叠加。

### Active / Pressed

按钮按下时允许 `transform: translateY(1px)`；选中筛选项应同时通过颜色和 `aria-pressed` 表达。

### Copy success

按钮文案切换为“已复制”；图标或背景可变为绿色，但不能仅用短暂 Toast。状态文本使用 `aria-live="polite"`。

## 19. 响应式规则

### ≥ 1180px

- 1200px 最大内容宽度；
- 12 栏栅格；
- 详情页双栏；
- 首页标题与代表工具卡可形成 7:5 构图。

### 768—1179px

- 页边距 24px；
- 首页首屏改为单栏；
- 详情侧栏下移；
- 目录控制区允许两行，但保持搜索框优先。

### ≤ 767px

- 页边距 16px；
- 导航文字缩小并允许滚动；
- Display 42px，H1 36px；
- 360px以下窄屏允许将Display降至36px，避免中文标题出现单字孤行；
- 大搜索框改为“输入框 + 下方全宽按钮”或保持单行但按钮仅用短文案；
- 卡片操作按钮全宽；
- Prompt 面板不产生页面级横向滚动；
- 所有触控目标最小 44×44px；
- Logo、面包屑、选择框和复制按钮也必须满足最小触控高度，不能因视觉紧凑缩小实际点击区域；
- 不通过隐藏重要信息解决拥挤，调整顺序与层级。

## 20. 动效

- 常规过渡 160ms，页面级切换 220ms；
- 缓动 `cubic-bezier(.2,.8,.2,1)`；
- 不使用循环浮动、光斑追踪、视差和滚动劫持；
- `prefers-reduced-motion: reduce` 时移除平移与平滑滚动。

## 21. Do / Don't

### Do

- 用真实版本、状态、格式和限制建立信任；
- 让搜索、筛选、排序和复制按钮具有真实行为；
- 首页、目录、详情和 About 使用同一套导航、间距和状态规则；
- 先保证中文阅读，再处理拉丁字母的品牌表现；
- 通过 Z 标志、蓝色结构线和深色 Prompt 面板形成识别度；
- 每次新增组件时优先复用本规范 token。

### Don't

- 不复制 SkillHub 的 Logo、吉祥物、文案或品牌视觉；
- 不复制 Linear、Intercom 等品牌的 DESIGN.md；
- 不添加虚构评分、下载量、用户数、认证或兼容环境；
- 不使用三种以上强调色；
- 不堆叠渐变、玻璃拟态、发光与大阴影；
- 不把首页做成后台控制台；
- 不用无行为的控件装饰页面；
- 不隐藏隐私、测试失败与已知限制。

## 22. 已知限制与待验证项

- 本规范基于当前单一公开工具形成；工具数量增长后，需要复验目录密度、排序和分页策略；
- Z 标志目前为文字型标志，尚未形成独立矢量品牌资产；
- 系统字体在 Windows、macOS、iOS 与 Android 上存在字重差异，需要跨设备复验；
- Any-to-MD `v0.1.0` 已完成 Codex 实际触发验证；
- 当前临时 Workers 地址含账户标识，不纳入品牌规范；
- 视觉小样已落实到正式 `app/` 页面并部署至临时 Workers 地址。

## 23. 实施检查清单

每次正式页面部署前，逐项确认：

- Logo 在 100% 和 125% 缩放下光学居中；
- 首页与目录路由独立；
- 搜索、筛选、排序、复制和下载均有真实行为；
- 键盘可完整访问；
- 桌面、平板和手机无横向溢出；
- 状态、版本、作者、许可证、隐私、限制与测试数据来自真实信息源；
- 页面发布信息统一读取 `app/tool-data.ts`，不在组件中重复硬编码版本和下载地址；
- `npm run validate:release` 已核对 ZIP、文件名、权威地址和 SHA-256；
- 不包含第三方品牌资产、专有字体或虚构数据；
- 构建、渲染测试和浏览器验收通过后，才进入部署流程。
