# z-skill for Codex

## 项目定位

`z-skill` 是周全个人 AI 工具发布站。网站用于介绍和交付经过设计、整理及验证的 Skill、Workflow、Agent 与 AI 工具，不扩展为文章站、社区、投稿平台或排行榜。

## 进入项目

- 开始修改前先读取 `README.md`、`DESIGN.md` 和当前 `git status`。
- 涉及公开工具、版本、状态、下载或安装 Prompt 时，以 `app/tool-data.ts` 和对应发布包为事实源。
- 保持真实版本、测试、隐私和限制，不编造评分、下载量、收藏量、用户数或兼容环境。
- 优先小步、可审查修改，不做无关重构，不覆盖或混入既有修改。

## 测试与 Git

- 按改动范围运行最快相关检查；正式发布前至少运行 `npm test` 和 `npm run lint`，并按 `DESIGN.md` 做桌面端和移动端浏览器验收。
- 唯一正式发布链路是 GitHub `main` → Cloudflare Workers Builds → `z-skill.com`；本项目不使用 Codex Sites，不得恢复 `.openai/hosting.json` 或新建 Sites 项目，除非用户明确确认迁移托管架构。
- 完成一次边界明确、检查通过的写入任务后，默认创建仅包含本轮文件的本地提交。
- 修改公开页面、公开下载包、候选版或正式版时，若用户已经确认“执行”“按方案执行”“发布”“上线”或“更新网站”，默认在测试与浏览器验收通过后提交并推送`main`，等待Cloudflare自动部署，再验收`z-skill.com`；上述确认视为包含本次公开交付，不要求用户另行重复说“推送”。
- 纯代码试验、方案草稿和尚未达到公开门槛的中间成果默认只做本地提交；用户明确说“先本地”“不要推送”或“不要部署”时必须停在本地。
- 写入前的计划必须明确说明本轮是“本地交付”还是“线上发布”；正式发布不得另行使用手工`wrangler deploy`绕过`main`自动构建链路。
- 不提交密钥、账号、本地缓存、依赖目录或构建中间产物。
