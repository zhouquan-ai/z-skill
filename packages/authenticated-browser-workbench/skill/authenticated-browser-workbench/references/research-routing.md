# 检索充分性与登录态渠道路由

## 先判断公开搜索是否足够

除非用户直接点名登录态网站、账号后台或浏览器交互，先使用AI自身的公开网页搜索能力。以下条件同时满足时直接完成，不启动BrowserSkill：

- 已经直接回答核心问题；
- 关键事实有当前、可访问且匹配问题的原始或权威来源；
- 不存在会改变结论的明显缺口或来源冲突；
- 用户不需要平台原生内容、真实体验、口碑或登录后数据。

## 自动升级条件

出现任一情形时使用本Skill，不要求用户再次点名平台：

- 公开结果主要是重复转载、聚合摘要、营销页面或过期资料；
- 关键页面需要登录，公开搜索只返回标题或片段；
- 问题需要评价、口碑、真实体验、避坑、服务过程或使用反馈；
- 需要平台内的近期讨论、机构回应、当事人动态或视频演示；
- 不同公开来源相互冲突，需要补充线索或交叉核验；
- 第一轮缺少有效关键词、候选对象或原始来源，需要已登录AI搜索网页扩展检索线索。

升级前明确“缺少什么”，只选择最可能补足缺口的一至两个渠道。

## 渠道模板

| 信息缺口 | 可配置渠道 | 证据定位 |
|---|---|---|
| 消费、生活服务、就医、旅行、产品使用等体验 | 生活方式或社区平台 | 用户经验和第三方线索 |
| 较完整的经验分析、专业讨论和争议观点 | 问答社区 | 观点与经验材料 |
| 最新动态、舆情、当事人或机构即时回应 | 即时社交平台 | 即时讨论和原始回应线索 |
| 操作演示、产品体验和较长视频讲解 | 视频平台 | 演示与体验材料 |
| 扩展关键词、候选来源和检索路径 | 已验证的登录态AI搜索网页 | 检索线索，不是独立证据 |
| 个人后台、账号内数据或登录后业务页面 | 对应目标站点 | 用户授权范围内的页面事实 |

使用者应在`site-matrix.md`中填写具体站点及当前验证状态。首选渠道受限时，再选择一个能补足同类缺口的替代渠道；不要遍历所有网站。

## 证据与停止条件

1. 使用官方网站、法规政策、原始文件和机构直接发布确认硬事实。
2. 使用平台帖子、评论和视频归纳体验、发现争议和形成待核实问题，不直接证明高风险事实。
3. 使用AI搜索网页扩展关键词和发现来源；尽量追溯其引用的原始链接。
4. 已补足主要缺口、继续搜索只会重复、同一路径一次修复和一次复测仍失败，或者只能转人工核实时，停止登录态检索。
5. 结果说明公开搜索和登录态检索分别贡献了什么，以及仍有哪些未核实事项。
