# 登录态浏览排障

## 扩展显示已连接，但命令找不到浏览器

运行 `bsk status` 和 `bsk browsers --json`。确认扩展开关、CLI与扩展协议版本、daemon状态和当前浏览器后台进程。一次无法修复时运行 `bsk doctor`，不要持续重启所有组件。

## 多个浏览器同时在线

关闭或停用非目标浏览器中的扩展。若后台旧连接暂时仍在，使用当前 `bsk browsers` 返回的实例显式启动目标会话。实例 ID 是临时标识，不写入长期配置。

## 页面框架可见但没有结果

等待几秒并重新 snapshot。懒加载站点可能先显示导航和占位结构。第一次结构为空不等于登录态失败。

## 新Agent标签页缺少登录态

先确认是否选错浏览器或账号。只有目标站点确实依赖现有页面上下文时，借用用户已经打开的目标标签页；不要通过浏览无关标签寻找登录态。

## selector或元素引用失效

导航或DOM变化后重新 snapshot。不要复用旧元素引用。在PowerShell中传递`@eN`引用时使用引号，避免被Shell解释。

## 提交成功但没有回答

分别记录输入、提交、生成和回答抽取。达到合理等待上限仍无输出时停止，改用站点矩阵中的替代渠道。不要用连续刷新或截图循环证明工具可用。

## 约30秒周期性断连

先在单浏览器、单扩展条件下用 `scripts/Test-BrowserSkillConnectionStability.ps1` 复现，记录CLI、扩展和浏览器版本。Manifest V3后台生命周期可能与断连有关，但不能只凭周期推断根因。公开候选版不自动复制或修改第三方扩展；优先核对上游新版本和已知问题。

## 验证码、风控和权限不足

请求用户接管验证码，不尝试绕过。出现风控后停止高频操作。账号到期或权限不足时只报告当前事实，不购买、续费或切换账号。

## 结束会话

无论成功或失败都执行上游BrowserSkill规定的会话停止命令。结束后运行 `bsk session list --json`；发现遗留会话时先清理，再开始新任务。
