# 已知限制

- 本包是 BrowserSkill 的外层编排模板，不包含 `bsk` CLI、浏览器扩展或上游 `browser-skill`，不能单独控制浏览器。
- BrowserSkill 官方当前明确支持 Chrome 和 Microsoft Edge；其他 Chromium 浏览器属于预期可用范围，需要逐机验证。
- 浏览器、扩展、CLI 和网站页面都会更新。候选版中的测试结论只对应 `TEST_MATRIX.md` 记录的日期与环境，不能保证未来版本保持一致。
- “扩展变绿”“页面可见”“输入成功”“提交成功”“取得回答”和“业务结果可信”是不同阶段，前一阶段通过不代表后一阶段通过。
- 多个浏览器实例在线时，必须现场运行 `bsk browsers` 并显式选择目标实例；历史实例 ID 不能作为长期配置。
- 某些站点会懒加载、要求验证码、触发风控或依赖现有标签页。模板不绕过验证码、权限控制或反自动化措施。
- 本包的稳定性脚本只在 Windows PowerShell 和 `bsk` CLI 环境验证；macOS、Linux、Chrome休眠唤醒和长时间任务尚未验证。
- 公开候选版不分发第三方扩展副本，也不自动修改 BrowserSkill。个人环境中出现的 Manifest V3 后台断连，应先复现、记录版本并向上游核对，不应把本地实验补丁当成通用修复。
- 站点矩阵是本地经验，不是平台兼容性承诺。公开模板默认不包含任何个人站点成功率或账号配置。
